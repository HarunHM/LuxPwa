Introduction To JavaScript

JavaScript was created by Brendan Eich in 1995 during his time at Netscape Communications. It was inspired by Java, Scheme and Self. JavaScript was initially developed for client-side and has been improved to server side programming language, also with the use of react native you can build Android and IOS Mobile application. It is an object based language which employes the use of object and methods in programming.

Remember: In JavaScripts objects are king, when you are good at Objects you are good in JavaScript.

Like other scripting language JavaScript is a light weigh, ie does not require a compiler since it is an interplated language. JavaScript mainly executes in the contex of a webpage,this is when the a client computer has a web browser which is javaScript enalbled.

SHORT HAND NOTES.

JavaScript being a dynamic programming language, it is widely accepted by most of the web browsers so that the interaction at client side with the user is made more interesting and more easier.
JavaScript also allows your page to be interactive. You can add animations into the page which either attract attention to a specific part of the page or which make the page easier to use.You can provide responses within the web page to various actions that your visitor takes so as to avoid the need to load new web pages to respond.
JavaScript is fast and Reliable. Functions can be rewritten and can be executed at the client side by using JavaScript.

JavaScript Syntax

JavaScript syntax refers to a set of rules that determine how the language will be written (by the programmer) and interpreted (by the browser). The JavaScript syntax is loosely based on the Java syntax. Java is a full blown programming environment and JavaScript could be seen as a sub-set of the Java syntax. Having said this, that is where the similarities end —Java and JavaScript are two totally different things.

JAVASCRIPT Syntax can be viewed as rules that a programmer has to follow to construct a program or software using Java script programming language.

JavaScript Comments

Comment are used as a techinique to explain code and making it more readable. In JavaScript, code after double slashes // or between /*and */ is not executed its treated as comments. Comment can be used to prevent part of a program from executing when testing possible code. In JavaScript there are two types of comment, that is single line and multiple line comments.
1) Single line comment. // comments
2) Mutiple line comments. /* comments
Comment are very important for documenting your code since they are ignored by browser and are not executed.

  • We use // (two slashes) for single line comments
  • We use /* comments */ for multiple line comments

JavaScript Variables

Variable are data containers. JavaScript variables are containers for storing data values which can change throughout the program. var keyword is used to declare variables. Equal sign (=) is assign a value to a variable, in JavaScript it is called assignment operator rather than eual to. In variable can be defined with or without a value.
A variable declared withou a value will have the value undefined

JavaScript variables must be indentified by a unique name called Identifiers.

Conventions For Naming Variables In JavaScript

  • You should not use Keywords naming indentifiers.
  • Indentifiers must begin with a letter or underscore but not a digit
  • JavaScript variable names are case-sensitive.
